The project helps developers learn and validate full-stack commerce architecture.

## Before Submitting

- Keep the repository public.
- Merge the branch that removes tracked runtime `.env` files.
- Confirm the README, license, mobile app docs, deployment docs, and CI workflow are visible on the default branch.
- Create a first GitHub release, for example `v0.1.0`, after the default branch is clean.
